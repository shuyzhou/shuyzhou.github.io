JavaScript判断数组的方法

2018-07-19 JavaScript

总结下JavaScript判断数组的方法以及其优缺点：

Array.isArray()

原生实现的方法，可以跨iframe判断。Array.isArray()本质上检测的是对象的 [[Class]] 值，[[Class]] 是对象的一个内部属性，里面包含了对象的类型信息，其格式为 [object Xxx] ，Xxx 就是对应的具体类型 。对于数组而言，[[Class]] 的值就是 [object Array] 。

instanceof

一般来说，利用instanceof可以判断一个对象是否是某个构造函数的实例，包括原生构造函数。但instanceof是通过原型判断，它假设只有一个全局执行环境。如果在有iframe的情况下，因为iframe中的Array和它所在的window的Array是两个构造函数，所以window的Array并不在iframe中的数组的原型链上，判断会出错。

Vue源码阅读（四）：diff算法

2018-07-19 JavaScript

接触Vue也有一段时间了，像我们这种数据可视化的产品通常有大量的数据要显示，所以经常用到v-for这个directive。而官方的教程中提到，建议给v-for循环出来的元素绑定key，而我有时候又会忘记绑定，导致时有bug，所以决定抽出点时间看看Vue的源码，搞清楚到底为什么要绑定key。

diff算法核心

当Vue检测到v-for相关的数组数据发生变化时，Vue先会利用diff算法对比变化前的数组和变化后的数组，然后确定对Dom的操作。diff算法分为两个阶段：初始化和更新。

初始化Dom

遍历数组元素，生成元素对应的Vue实例，将实例存入到缓存中，并将实例对应Dom元素插入到document中。

更新

遍历数组元素，从缓存中查找是否已存在元素对应的实例，有的话标记该实例为可复用。遍历旧数组，从缓存中删除数组中不被复用的实例，并删除相应Dom元素。复用这些标记实例对应的Dom元素，并调整元素顺序，再将数组中新增的数据对应的Dom元素插入到document中。
关键就在，这个缓存其实是一个对象，它是v-for数据对应的实例的集合。实例的key的值是Vue按照一定规则得到的，如果用户给该实例绑定了key，那么key值就是绑定的表达式的值；如果用户没有绑定了key，那么key一般是该实例在数组中对应的下标值。所以，如果没有指定key的话，那么随着数组的变化元素的下标也产生了变化，导致从缓存中得到的是错误的实例，从而导致显示的bug。比如数组从[a,b,c]变化成[b,c]时，明明删除的是第一个元素a，但是由于b对应的下标变成了0，c的变成了1，结果被复用的是a和b对应的Dom元素。

ES6的迭代协议

2018-07-08 JavaScript

可迭代协议

如果一个Javascript对象遵循可迭代协议，那么这个对象是可迭代的，可以在例如for..of等一些操作中被遍历。一个可迭代的对象，必须实现@@iterator方法，即对象或者对象原型链上某个对象必须有一个属性名为Symbol.itertor的无参函数，这个函数返回的对象遵守迭代器协议。当一个对象需要被迭代时，它的@@iterator会被调用，然后返回一个在迭代中获得值的迭代器。

迭代器协议

迭代器对象实现了next方法，next方法不接受参数，返回的对象拥有done和value两个属性。当迭代未完成时，done为false,value为任意Javascript值；当迭代完成时，done为true,value的值可省略。

相关的内置对象、API和语法

  • 内置可迭代对象
    String、Array、TypedArray、Map和Set时所有内置可迭代对象
  • 用于可迭代对象的语法
    for..of、spread、yield*和结构操作符
  • 接受可迭代对象的内置API
    Map、WeakMap、Set和WeakSet构造函数，以及Promise.all()、Promise.race()、Array.from()方法

当Generator生成器被执行时，写在函数里的代码并不会立即执行，而是会被包装成一个Generator对象并被返回。Generator对象共有三个实例方法：next、return和throw。

Generator.prototype.next()

当next方法每次被调用时，生成器函数会从函数最开始或上一个yield语句之后开始执行，只到下一个yield语句或者函数结束。next方法返回对象的value为yield语句后面的表达式的值或者是函数的返回值。

next方法可以带参数，参数将作为yield语句的返回值，如果是第一次执行，该参数将被忽略。

Generator.prototype.return()

当return方法被调用时，Generator对象将提前结束迭代并将返回值置为接受的参数。当一个已经结束的Generator对象调用return方法时，它的返回值将会被置为传入的参数，已结束的状态不变。

Generator.prototype.throw()

throw方法向Generator函数内部传入一个错误，并继续下一次迭代，返回值和next函数一样。

JavaScript中Symbol类型

2018-07-08 JavaScript

在ES6之前，JavaScript中一共有六种数据类型：undefined、null、boolean、string、number和object，而ES6中新增了一种基本数据类型symbol。symbol值可由Symbol函数产生，并且Symbol拥有一些内置的symbol类型的静态属性（例如Symbol.iterator）。

symbol基本语法和用法

Symbol函数不支持语法new Symbol()，每一个从Symbol()返回的symbol值都是唯一的，即使两次Symbol()接受的参数是同一个参数，返回的也不是同一个symbol值。symbol值可以用来作为对象属性的标识符，该属性不会被for..of和Object.getOwnPropertyNames返回，但会被Object.getOwnPropertySymbols返回。看起来，这是为了解决属性名冲突而存在的类型。当有两个以上的库对同一个对象做访问或修改时，难以做到某个库定义的内部属性和方法不会被其它库访问或修改，而symbol可以解决该问题。

Symbol的静态方法

  • Symbol.for(key)
    使用给的key搜索现有的symbol，如果找到则返回给symbol。否则将使用给定的key在全局symbol注册表中创建一个新的symbol，并返回该symbol。
  • Symbol.keyFor(sym)
    从全局symbol注册表中返回该symbol的key。
