雅虎前端性能优化原则之总结

2018-07-31 性能

本文是对于雅虎前端性能优化原则的简单总结，原文请移步这里。

减少HTTP请求次数

合并js、css文件

合并文件，体积不会发生变化，但相应的HTTP请求减少。

CSS Sprites

将多张背景图片合并成一张图片，利用CSS的background-image和background-position属性显示相应背景，缺点是添加图片或改变原有图片大小都必须重新合成图片，并修改代码。

Image maps

这个没看太懂，大概意思是一个页面有多张连起来的图片可以合并成一张图片？

内联图片

现在一种很常见的方式是对一些体积小的图片（通常是图标之类的）进行base64编码，利用Data url将编码后的字符串嵌入到HTML中或CSS中。因为图片体积不大，所以并不会对嵌入后的文件加载时间造成很大影响。

利用CDN(Content Delivery Network)

有时会由于物理距离过远，导致资源加载时间很长，这种问题从软件上是无法解决的，只能在离用户较近的地方架设缓存服务器，这就是CDN。CDN中一般有多台服务器形成一个覆盖这世界上绝大多数地区的网络，一个应用的静态资源（一般是前端的一些资源）可以缓存在这些缓存服务器中，当用户访问一个域名时将自动从离用户最近的服务器中获取资源，这样将节省资源加载时间。

添加Expires或Cache-Control头

缓存静态资源提高页面加载效率，缓存分为强制缓存和协商缓存，利用Expires或Cache-Control头可以设定资源缓存的寿命，在缓存未过期之前对资源进行强制缓存。如果在缓存未过期之前你需要对资源进行更新的话，则必须改变文件名。

Gzip压缩响应

HTTP/1.1定义，Accept-Encoding请求头说明客户端接受的编码方式，最常见的如Accept-Encoding:gzip, deflate，而服务器根据该请求头选择一种编码方式对响应进行压缩，并将压缩方式返回在响应头中：Content-Encoding: gzip。

样式表链接放在head，脚本链接放在body底部

浏览器在解析body部分后开始渲染页面，样式表的加载不会阻塞浏览器渲染页面，所以样式表应该被尽快加载和解析，否则渲染出的页面将是无样式的；相反，脚本的加载和执行将阻塞其它资源加载和页面渲染，所以需要把脚本放在body底部，令它最后执行。在最新的标准中，<script>标签定义了defer属性和async属性，都是让脚本并行下载，但是defer下脚本按照顺序执行，而async不按顺序执行脚本。

避免CSS表达式

在浏览页面时，即使用户的操作并不影响到样式的改变（比如移动鼠标），CSS表达式也会被重复计算很多次，这对页面性能有一定影响。

利用单独的文件替代内联脚本和样式

这使得脚本和样式可以被缓存起来，而不是每次都随着HTML一同被下载。

减少DNS查找次数

抛开缓存的因素，一个页面的资源有多少个域名就会产生多少次DNS查找。DNS查找需要一定时间，而减少DNS查找次数可以一定程度上节约响应时间。但域名同样影响到资源的并行下载，根据HTTP/1.1，浏览器最多同时加载两个相同域名的资源，而不同域名的资源则无此限制。所以推荐域名的个数在2-4个之间。

压缩JavaScript和CSS

减少资源体积，节省带宽。

避免重定向

重定向会比普通请求要多一次请求，避免重定向将减少请求次数。

删除重复脚本

删除页面中重复引用的脚本，以免造成重复的请求或执行。

设置Etag

Etag是某个资源某个特定版本的标识符，浏览器利用它和If-None-Match头以及Last-Modified和If-Modified-Since来和服务器进行协商缓存。Etag存在一个问题就是当存在多台服务器时，不同服务器上某个资源同版本的Etag可能不同，这时就失去了它原有的利用缓存的功能。

缓存Ajax结果

这在某些情况下能提高一定程度的性能，但当数据变化时如何有效清除缓存是个问题。雅虎提出了一种方案，就是给请求的url添加一个参数（比如&t=1190241612），每次需要从服务器获取数据而不是缓存时就改变该参数值，但也只适用于小部分情况，并不能成为一种通用的解决办法。

尽早刷新文档的输出

这个大概意思是说，以PHP为例，服务器处理HTML请求时不用全部拼接完再响应，而是先响应处理好的一部分（比如头部），然后在等待服务器拼接剩余部分时浏览器可以先加载头部中链接的资源。

拆分初始化负载

页面加载时只加载初始化页面的资源，其它资源等初始化完成之后加载，可以减少渲染时间。

预加载资源

预加载接下来可能会需要的资源，能提高用户体验。

划分主域

利用浏览器不同域的资源可以并行加载的特点，可以适当将静态资源放在不同域下，提高资源加载速率。

Ajax尽量使用GET方法

当使用XMLHttpRequest对象时，POST方法会发送两个包（头部和数据），而GET方法一般是一个包。当需要使用Ajax从服务器获取数据而不是发送数据时，尽量使用GET方法。

减小Cookie体积

Cookie随着请求和响应发送，减小Cookie体积也就相应减少了请求和相应的体积。

图片优化

可以对图片进行压缩，减少图片体积。

Array.isArray()

原生实现的方法，可以跨iframe判断。Array.isArray()本质上检测的是对象的 [[Class]] 值，[[Class]] 是对象的一个内部属性，里面包含了对象的类型信息，其格式为 [object Xxx] ，Xxx 就是对应的具体类型 。对于数组而言，[[Class]] 的值就是 [object Array] 。

instanceof

一般来说，利用instanceof可以判断一个对象是否是某个构造函数的实例，包括原生构造函数。但instanceof是通过原型判断，它假设只有一个全局执行环境。如果在有iframe的情况下，因为iframe中的Array和它所在的window的Array是两个构造函数，所以window的Array并不在iframe中的数组的原型链上，判断会出错。

Vue源码阅读（四）：diff算法

2018-07-19 JavaScript

接触Vue也有一段时间了，像我们这种数据可视化的产品通常有大量的数据要显示，所以经常用到v-for这个directive。而官方的教程中提到，建议给v-for循环出来的元素绑定key，而我有时候又会忘记绑定，导致时有bug，所以决定抽出点时间看看Vue的源码，搞清楚到底为什么要绑定key。

diff算法核心

当Vue检测到v-for相关的数组数据发生变化时，Vue先会利用diff算法对比变化前的数组和变化后的数组，然后确定对Dom的操作。diff算法分为两个阶段：初始化和更新。

初始化Dom

遍历数组元素，生成元素对应的Vue实例，将实例存入到缓存中，并将实例对应Dom元素插入到document中。

更新

遍历数组元素，从缓存中查找是否已存在元素对应的实例，有的话标记该实例为可复用。遍历旧数组，从缓存中删除数组中不被复用的实例，并删除相应Dom元素。复用这些标记实例对应的Dom元素，并调整元素顺序，再将数组中新增的数据对应的Dom元素插入到document中。
关键就在，这个缓存其实是一个对象，它是v-for数据对应的实例的集合。实例的key的值是Vue按照一定规则得到的，如果用户给该实例绑定了key，那么key值就是绑定的表达式的值；如果用户没有绑定了key，那么key一般是该实例在数组中对应的下标值。所以，如果没有指定key的话，那么随着数组的变化元素的下标也产生了变化，导致从缓存中得到的是错误的实例，从而导致显示的bug。比如数组从[a,b,c]变化成[b,c]时，明明删除的是第一个元素a，但是由于b对应的下标变成了0，c的变成了1，结果被复用的是a和b对应的Dom元素。

ES6的迭代协议

2018-07-08 JavaScript

可迭代协议

如果一个Javascript对象遵循可迭代协议，那么这个对象是可迭代的，可以在例如for..of等一些操作中被遍历。一个可迭代的对象，必须实现@@iterator方法，即对象或者对象原型链上某个对象必须有一个属性名为Symbol.itertor的无参函数，这个函数返回的对象遵守迭代器协议。当一个对象需要被迭代时，它的@@iterator会被调用，然后返回一个在迭代中获得值的迭代器。

迭代器协议

迭代器对象实现了next方法，next方法不接受参数，返回的对象拥有done和value两个属性。当迭代未完成时，done为false,value为任意Javascript值；当迭代完成时，done为true,value的值可省略。

相关的内置对象、API和语法

  • 内置可迭代对象
    String、Array、TypedArray、Map和Set时所有内置可迭代对象
  • 用于可迭代对象的语法
    for..of、spread、yield*和结构操作符
  • 接受可迭代对象的内置API
    Map、WeakMap、Set和WeakSet构造函数，以及Promise.all()、Promise.race()、Array.from()方法

当Generator生成器被执行时，写在函数里的代码并不会立即执行，而是会被包装成一个Generator对象并被返回。Generator对象共有三个实例方法：next、return和throw。

Generator.prototype.next()

当next方法每次被调用时，生成器函数会从函数最开始或上一个yield语句之后开始执行，只到下一个yield语句或者函数结束。next方法返回对象的value为yield语句后面的表达式的值或者是函数的返回值。

next方法可以带参数，参数将作为yield语句的返回值，如果是第一次执行，该参数将被忽略。

Generator.prototype.return()

当return方法被调用时，Generator对象将提前结束迭代并将返回值置为接受的参数。当一个已经结束的Generator对象调用return方法时，它的返回值将会被置为传入的参数，已结束的状态不变。

Generator.prototype.throw()

throw方法向Generator函数内部传入一个错误，并继续下一次迭代，返回值和next函数一样。

Generator不可重复使用
