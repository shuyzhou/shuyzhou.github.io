ES6的迭代协议

2018-07-08

可迭代协议

如果一个Javascript对象遵循可迭代协议，那么这个对象是可迭代的，可以在例如for..of等一些操作中被遍历。一个可迭代的对象，必须实现@@iterator方法，即对象或者对象原型链上某个对象必须有一个属性名为Symbol.itertor的无参函数，这个函数返回的对象遵守迭代器协议。当一个对象需要被迭代时，它的@@iterator会被调用，然后返回一个在迭代中获得值的迭代器。

迭代器协议

迭代器对象实现了next方法，next方法不接受参数，返回的对象拥有done和value两个属性。当迭代未完成时，done为false,value为任意Javascript值；当迭代完成时，done为true,value的值可省略。

相关的内置对象、API和语法

  • 内置可迭代对象
    String、Array、TypedArray、Map和Set时所有内置可迭代对象
  • 用于可迭代对象的语法
    for..of、spread、yield*和结构操作符
  • 接受可迭代对象的内置API
    Map、WeakMap、Set和WeakSet构造函数，以及Promise.all()、Promise.race()、Array.from()方法

当Generator生成器被执行时，写在函数里的代码并不会立即执行，而是会被包装成一个Generator对象并被返回。Generator对象共有三个实例方法：next、return和throw。

Generator.prototype.next()

当next方法每次被调用时，生成器函数会从函数最开始或上一个yield语句之后开始执行，只到下一个yield语句或者函数结束。next方法返回对象的value为yield语句后面的表达式的值或者是函数的返回值。

next方法可以带参数，参数将作为yield语句的返回值，如果是第一次执行，该参数将被忽略。

Generator.prototype.return()

当return方法被调用时，Generator对象将提前结束迭代并将返回值置为接受的参数。当一个已经结束的Generator对象调用return方法时，它的返回值将会被置为传入的参数，已结束的状态不变。

Generator.prototype.throw()

throw方法向Generator函数内部传入一个错误，并继续下一次迭代，返回值和next函数一样。

ES6中的Promise对象

2018-07-05 JavaScript

解决的问题

由于浏览器中JavaScript的执行是单线程的，所以所有的网络操作、浏览器事件都是异步的，通过回调函数来执行的。当回调函数嵌套很深时，代码的可读性将下降，Promise就是为了解决这个问题而出现的。

基本概念

使用new Promise(fn)的语法会得到一个Promise对象，它的参数（通常是一个异步函数）将会被立刻执行。Promise对象有三种状态：Pending（进行中）、Resolved（已完成，又称 Fulfilled）和 Rejected（已失败），状态的转变只有两种可能：从 Pending 变为 Resolved 和从 Pending 变为 Rejected，而什么时候进行什么转变取决于异步函数的执行结果。
Promise基本语法如下：

当一个Promise对象被resolve掉时，它的状态将变成resolved，回调函数的参数就是resolve方法的参数。当它被reject掉时，它的状态将变为rejected，回调函数的参数则是reject方法的参数。当对一个Promise对象执行then方法时，如果它是pending状态，则在它状态变为resolved或rejected时将回调函数放入js的执行队列，如果它已经是resolved或rejected状态时则立刻将回调函数放入执行队列。

基本API

1.Promise.prototype.then()

then方法是Promise对象的实例方法，用来给Promise对象定义回调函数，接受参数为两个函数：Promise的成功和失败情况的回调函数。then方法的返回值同样是一个Promise对象，这个对象的状态和回调函数的返回值有关。
简单来说，如果回调函数返回的是一个Promise对象，那么then方法返回的Promise对象状态与之同步，回调函数的参数也是一致的；如果回调函数返回的是一个值，那么then方法返回的Promise对象状态为resolved；如果回调函数抛出了一个错误，那么then方法返回的Promise对象状态为rejected。当对应某个状态的回调被省略或为非函数时，then方法会丢失该状态的回调函数，但不会产生错误。

2.Promise.prototype.catch()

catch方法是Promise对象的实例方法，用来给Promise对象定义失败是回调函数，是then(null, ...)的别名。它的返回值同样是一个Promise对象。

3.Promise.resolve()

resolve方法返回一个状态为resolved的Promise对象，接受的参数将作为该Promise对象的值；如果接受的参数是thenable的（该参数有then方法），则Promise将采取该对象的最终形态作为该Promise对象的值；如果接受的参数为Promise对象，resolve方法将返回该Promise对象。

4.Promise.reject()

reject方法返回一个状态为rejected的Promise对象，接受的参数将作为该Promise对象的值

5.Promise.all()

该方法接受参数为可迭代的对象（例如数组），返回一个Promise对象。当该数组里所有Promise对象resolve时，该Promise对象的状态变为resolved；若有其中一个被reject，该Promise对象的状态变为rejected。

6.Promise.prototype.finally()

不管调用finally的Promise对象状态为resolved或rejected，该方法的回调函数都会被调用，且回调函数不接受参数。

Vue源码阅读（二）：数据监听

2017-06-17 JavaScript

当刚开始看源码的时候，很多人都会觉得源码很难看懂，很容易让人想要放弃。对使用者来说，源码就像一个黑盒子，我们只会用它的功能，却不知道它的内部是怎么实现这些API的。所以，我们不妨先抛出一些相关问题：

  1. Vue怎么对输入的数据进行代理的？
  2. Vue怎么监听数据变化的？
  3. 通过$data获取原始数据或重置数据是怎么实现的？

接下来，让我们从源码中寻找答案。

对输入数据进行代理

vue实例有一个内部属性_data，用来保存该实例的数据。vue通过_proxy方法绑定对_data的代理。

在_setData方法中，将新的data赋值给实例的_data属性。比较新的data和旧的data，对需要删除的属性解除代理，对于需要新增的属性绑定代理。接着，断开旧data对该实例的引用，调用observe方法对新的data进行监听，最后调用实例的_digest方法强制更新所有watcher。

对象池

游戏中一共三类角色，主角、爱心和大便（…），接到一个爱心加一分，被大便砸到失去一条命。为了方便地管理角色已经维护代码，这些角色的状态和更新方法都存放着对应的实例对象中。

爱心和大便会一直从canvas顶部掉下，如果每次都创建一个新对象的话会在短时间内创建了大量的对象，这样将消耗大量资源。而这些对象的方法都是相似的，只有一些参数不一样，显然可以利用对象池对对象进行回收。大致思想是定义一个数组存放所有对象，当外界来“取”对象时从数组尾部弹出一个未激活对象压入数组头部，调用该对象的spawn方法，并将该对象标记为激活状态。在每一次render函数执行时，对象池将更新所有已激活对象的坐标，调用每个已激活对象的draw方法。并且，当某对象和主角碰撞或到canvas底部时对象池将对象压入数组尾部，并调用该对象的reset方法。

为了判断爱心有没有被接住，需要对两个对象进行碰撞检测。碰撞检测大致分为两步：

  1. 计算两个矩形重叠的矩形坐标
  2. 进一步检测像素级碰撞

第2步中首先创建一个新的canvas对象，在该对象中依次画出两副图像在重叠矩形部分的图片块，并利用getImageData方法获取其像素数据，最后判断两幅图片有没有在重叠的像素。
